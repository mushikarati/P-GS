🐠 Emoji‐Based Programming Languages
ojicode (2015) – An open‐source, strongly typed programming language whose syntax is entirely composed of emoji .  It supports modern features (objects, generics, closures, etc.) and even appears in university courses (e.g. Harvard CS50) .
•	Egaki (2023) – A hobby language by Anthony Volk, implemented in Python, with an “entirely emoji-driven syntax” .  Volk describes Egaki as a parser/interpreter built to let developers program using only emoji.
•	EmojiScript (2023) – A Node.js/JavaScript language where emojis serve as tokens.  For example, the arrow symbol ➡️ is used for assignment (a ➡️ 10 sets a = 10) , and ❗ ends a print statement. This language is distributed via npm (the-emojiscript) and comes with an online REPL/editor .
•	Emojiscript (2024) – A Go-based project (Pennanen) that “accepts only emoji as input” .  It’s a minimalist esolang inspired by Thorsten Ball’s Monkey interpreter tutorial; documentation is sparse but it exists as a GitHub project.
•	Emojinal (2014) – A conceptual all-emoji language by Ramsey Nasser (FastCompany article).  It was designed as an art experiment (stack-based, no standard punctuation) rather than a practical tool .  Nasser notes such languages carry cultural assumptions and are more “statement” than widely used languages.
•	Emoji (esolang, 2015) – A documented esoteric programming language where each emoji acts as a stack operation.  It’s purely a toy language: for example, “Hello World” is encoded entirely with emoji commands .  (There’s even a Python interpreter on GitHub for it.)
•	“emojilang” (GitHub) – A Python project implementing an emoji-language using the Lark parser.  Its grammar maps emoji to familiar operators: e.g. ➕ for +, ✖ for *, ✔/❌ for boolean constants, etc. (see the syntax table ).  This shows other hobbyists building full emoji grammars in Python.

•	EmojiLM (2023) – A recent arXiv study treats emoji usage as a “new emoji language” .  Peng et al. synthesized a large text–emoji parallel corpus and trained a seq2seq model (EmojiLM) for bidirectional translation between text and emoji.  This work highlights serious NLP interest in modeling emoji as linguistic tokens.
•	Emoji Grammar (2019) – Cognitive scientists McCulloch and Gawne analyzed whether emoji have a grammar.  They found that people do use emoji in communication, but the sequencing rules are very simple.  In fact, emoji often act like nonverbal cues or punctuation: participants typically placed emoji at sentence ends to add tone, akin to “emotionally charged punctuation” .  The study concludes that any internal grammar of emoji is quite limited in complexity .
•	Punctuation and Emotion (NLP) – In text analytics, it’s well-known that punctuation conveys sentiment.  For example, Liu et al. note that punctuation can express emotion (e.g. “We are late! Hurry up!!!” uses exclamation marks for urgency) .  Most sentiment‐analysis tools (NLTK, VADER, etc.) do incorporate simple rules for repeated “!!!” or “???” to boost sentiment scores.  However, these remain statistical heuristics. By contrast, the Codex system is unusual in building a purely symbolic, explainable grammar over punctuation (no machine learning).
